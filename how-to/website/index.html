How To Make a Website

So, you want to make a website? Here's what you need to consider:

  • Content - The content of your website
  • Hosting - Where your website is hosted
  • Structure - The file/folder structure of your website
  • Styling - The styles of your website

Content

This is, obviously, the most important thing to consider when making a website. Usually. I suppose you could just be making one as a technical experiment, in which case I guess "Lorem ipsum dolor sit amet" will do.

Hosting

For people to access your website it must be hosted on a web-server. This server delivers the pages of your site to the people accessing it, and in some cases may also do some server-side-processing, depending on the site and the server involved.

I personally use GitHub Pages to host this website, because it doesn't need any server-side-processing, which GitHub Pages doesn't provide. It also allows you to use a "projectname.github.io" domain name or your own top level one. There are many different hosting options available, at different costs and offering different things, and it is definitly worth looking around to find the right option for you. Also, if you are going to use your own domain, take your time to check all the different name registrars to find the best price, as they often change.

Structure

The file/folder structure of your website is very important. The root folder is the one that is accessed at your domain name, e.g. www.msudbury.uk provides the highest folder level, and there is no way to access the files outside this folder, but then there is www.msudbury.uk/gallery, which is a folder within the root. Most web servers will send the user an 'index' file when accessing a folder, so when you go to www.msudbury.uk, you are being shown www.msudbury.uk/index.html.

It is a good idea when planning a website to draw out a 'site-map'. This can be done on a piece of paper, or within a computer program, and it allows you to see how different files can be accessed. When I make a website, I ususally give each page its own directory. One reason for this is that it allows you to give each one files of the same name, e.g. each page could have it's own stylesheet.css file, whereas if all pages were in the same directory those files would have to have different names.

Styling

The styling of your website is also very important, as this has a large impact on how your content is percieved. Your styling consists of rules, e.g. 'background-color: #99ff99', which are applied to elements, e.g. 'body'. There are many more complex things which can be achieved with CSS, and using an external stylesheet (a .css file), as I do on this site, allows the same rules to be applied to multiple pages on the same site.
